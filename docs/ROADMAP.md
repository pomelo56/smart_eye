# 智能识别演进路线

> 慧眼 SmartEye 从规则引擎到 AI 理解的技术演进规划。

## 当前架构（v0.8.3）：规则引擎 + 应用内更新

```
相机画面 → ML Kit OCR → 正则匹配取餐码 → 硬编码平台规则 → 预录音频拼接 → 播报
                                      ↓
                            应用内更新检查（本地版本 vs 远程最新版）
```

**局限**：每新增一种小票格式或物品类型，都要手动改代码、录音频、发版。

## 演进目标

让系统**自己理解** OCR 文本，无需预设规则即可识别任意外卖小票格式，并逐步扩展到非文字物品。

## Phase 2：云端 LLM 语义理解（v1.0.0）—— 已冻结

### 核心思路

在现有规则引擎之上，增加一个 `AiParser` 层：

```
相机画面 → ML Kit OCR → 快速匹配（已知平台，0ms）
                     → AiParser（未知格式，1-2s）→ 国产 LLM API → 结构化JSON → TTS播报
```

- 已知平台（美团/饿了么/京东等）走本地正则，**零延迟、零成本**
- 未知格式调云端 LLM，**自动兼容任何新平台**
- 没网时 fallback 到本地规则，不会完全不可用

### AiParser Prompt 示例

```
你是视障辅助工具的文本解析模块。从以下 OCR 文本中提取结构化信息。

规则：
1. 如果文本包含外卖取餐码（如 #65、美团-261），提取 code 和 platform
2. 如果文本是其他物品标签（如药盒、遥控器），提取 item_name 和 key_info
3. 如果无法判断，返回 type: "unknown"

OCR文本："美团-261 配送中 2026-07-05"
返回JSON（不要其他内容）：
{"type":"meal_pickup","platform":"美团外卖","code":"261"}
```

v1.0.0 是 SmartEye 的**正式版里程碑**，意味着从"公益原型"进入"可公开分发、可长期维护"的产品阶段。在未完成 `VERSION.md` 中的 v1.0 readiness checklist 并由核心维护者人工批准前，本阶段**不允许进入开发排期**。

### 额度管理

| 场景 | 行为 |
|------|------|
| 有网 + 本地正则匹配 | 不调 API，零成本 |
| 有网 + 本地未匹配 | 调用 LLM API |
| API 调用失败 | Fallback 到本地规则，播报识别到的号码（无平台） |
| API 额度耗尽 | 同上，播放提示"网络识别服务暂不可用" |
| 无网络 | 纯本地规则，不调 API |

每日预计调用次数：盲人日常点外卖 1-3 次，每次多码场景最多 3 次调用。按最保守估计每日 < 10 次，远低于任何免费额度。

### 实施改动

- 新增 `lib/services/ai_parser.dart`：HTTP 调用 LLM API，解析 JSON 响应
- 修改 `lib/screens/home_screen.dart`：OCR 后先走本地，失败调 AiParser
- 新增 `lib/services/tts_system.dart`：尝试系统 TTS，失败 fallback 到音频拼接
- 配置页面：用户输入自己的 API Key（不硬编码到 APK 中）
- APK 增量：约 50KB（http 包）

## Phase 2.5：系统语音合成替代预录音频（v1.1.0）

### 核心思路

当前方案每新增一个词（平台名、位置词等）都要录音频、打 APK 包。

用 Android 系统 TTS 替代：
- LLM / 规则引擎返回纯文本 → 系统 TTS 直接朗读
- OPPO/ColorOS 上系统 TTS 不可用 → fallback 到预录音频拼接
- 优点：任何新平台名、新格式都能直接朗读，不用维护音频库

### 改动

- 新增 `lib/services/tts_system_service.dart`：封装 Android 系统 TTS
- 修改 `TtsService`：优先尝试系统 TTS，失败时 fallback 到音频拼接
- APK 增量：约 20KB

## Phase 3：多模态云端理解（v2.0.0）

### 核心思路

跳过 OCR，直接把相机画面发给多模态大模型（如智谱 GLM-4V-Flash，永久免费）：

```
相机画面 → VLM → "画面中央有一张外卖小票，取餐码65号，美团外卖"
                → "桌面上有一个黑色遥控器，品牌是小米"
                → "面前有一瓶药，标签上写着布洛芬，每次一片"
```

### 能力扩展

- 识别非文字物品：药盒、遥控器、商品标签、门牌号等
- 描述物体位置："遥控器在桌子右侧"
- 阅读复杂文本：信件、说明书、菜单

### 成本

- 图片比文本贵，单次约 1000-2000 tokens
- 免费额度下每日约 2500-5000 次，仍远超需求
- 可做高级功能开关（默认用 Phase 2 的文本方案，用户选择开启图片识别）

## 小票导航功能

### Phase A：位置记忆引导（v0.6.0，已实现）

当小票移出画面时，根据上次识别到取餐码的位置，语音引导用户重新对准。

```
1. 识别到取餐码 → 记住位置（如"左下"）
2. 画面完全无文字 → "小票不在画面中，请向下方移动手机"
3. 有文字但无取餐码 → "识别中，手机请稳一些"（已有功能）
4. 重新对准 → 正常播报
```

- 5 秒冷却防止重复播报
- 三击重置时清除位置记忆
- 播报期间暂停引导

### Phase B：差分帧检测（v0.8.0，计划中）

对比前后两帧的画面差异，判断小票移出的方向：
- 上一帧有文字，这一帧没有 → 小票从画面中消失了
- 消失前小票在画面右侧 → 提示"请向右移动"
- 加入距离估算：文字大小变化 → "再近一点" / "再远一点"

### Phase C：物体检测追踪（v2.0.0，远期）

使用 ML Kit ObjectDetector 检测并追踪画面中的"receipt-like"物体：
- 实时追踪小票在画面中的位置
- 即使没有取餐码也能引导对准
- 小票部分入画时提示"再往左一点"
