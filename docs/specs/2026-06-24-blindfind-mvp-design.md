## 1. 项目概述

### 1.1 项目名称
盲寻（BlindFind）—— 盲人视觉辅助小工具集

### 1.2 项目性质
公益性项目，免费开源

### 1.3 目标用户
- 全盲中青年用户
- 有较丰富的智能手机使用经验
- 主要使用安卓设备

### 1.4 核心痛点
集体取外卖场景下，盲人用户无法识别哪一份外卖是自己的，需要他人帮助。

### 1.5 MVP 目标
实现一个可在本地运行的外卖查找辅助工具，通过摄像头实时检测外卖、提供寻路提示、识别外卖单上的关键信息（取餐人、取餐码等），并用语音播报给用户。

---

## 2. 技术路线

### 2.1 两阶段开发策略

| 阶段 | 平台 | 目标 | 技术栈 |
|------|------|------|--------|
| 第一阶段 | Mac (Python) | 算法验证，跑通完整流程 | Python + OpenCV + PyTorch + PaddleOCR |
| 第二阶段 | 移动端 (Flutter) | 移植到安卓，用户可用 | Flutter + tflite_flutter + 系统TTS |

### 2.2 为什么分两阶段
- AI应用最大风险是模型效果不达预期，先在Python环境快速验证
- Mac上调试效率高，模型选择自由
- 算法验证通过后再做移动端移植，风险可控

## 3. 第一阶段（Mac端）详细设计

### 3.1 功能清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 外卖检测 | 实时检测画面中的外卖盒/外卖袋/小票 | P0 |
| 避障寻路提示 | 检测障碍物，语音引导方向 | P0 |
| OCR文字识别 | 识别外卖单上的取餐人、取餐码、商家 | P0 |
| 语音播报 | 用系统TTS朗读识别结果 | P0 |
| 实时画面预览 | 显示检测框，方便调试 | P1 |

### 3.2 技术选型

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 编程语言 | Python 3.10+ | AI生态成熟 |
| 摄像头采集 | OpenCV (cv2.VideoCapture) | 跨平台，简单直接 |
| 物体检测 | YOLOv8n (Ultralytics) | 轻量高速，Mac M5可实时运行 |
| OCR识别 | PaddleOCR PP-OCRv4 轻量版 | 中文效果好，有预训练模型 |
| 语音播报 | pyttsx3 或 macOS `say` 命令 | 调用系统TTS，离线 |
| 画面展示 | OpenCV imshow | 画检测框，调试用 |
| 深度学习框架 | PyTorch + PaddlePaddle | YOLO用PyTorch，OCR用Paddle |

### 3.3 模型方案

#### 3.3.1 外卖检测模型：YOLOv8n
- **模型大小**：约6MB（nano版）
- **推理速度**：Mac M5 上预估 30-50 FPS（MPS加速）
- **检测策略**：
  - 先用 COCO 预训练模型，检测"背包/手提包/瓶子/书"等近似类别
  - 后续根据需要用外卖场景数据微调
- **输出**：检测框坐标 (x1, y1, x2, y2)、置信度、类别

#### 3.3.2 OCR模型：PaddleOCR PP-OCRv4 轻量版
- **模型大小**：检测+识别共约10MB
- **特点**：中文识别效果好，支持竖排文字
- **识别内容**：
  - 取餐人姓名
  - 取餐码/手机号尾号
  - 商家名称
- **触发时机**：检测到外卖且用户确认后，才触发OCR（节省性能）

#### 3.3.3 避障方案：复用YOLOv8n
- 不单独使用深度估计模型
- 用同一个YOLO模型检测常见障碍物：人、椅子、桌子、背包、瓶子等
- 通过检测框的大小和位置粗略判断距离和方向

### 3.5 核心流程

#### 3.5.1 外卖查找主流程

```
启动 → 摄像头初始化 → 模型加载
    │
    ▼
主循环：
  1. 采集一帧图像
  2. YOLO检测（外卖 + 障碍物）
  3. 检测到外卖？
     ├── 是 → 计算位置和距离 → 语音播报（带节流）
     └── 否 → 静默
  4. 检测到障碍物？
     ├── 是 → 判断方向 → 语音播报（高优先级）
     └── 否 → 无动作
  5. 用户按键/语音触发OCR？
     ├── 是 → 截取当前帧 → PaddleOCR识别 → 提取关键字段 → 语音播报
     └── 否 → 继续循环
```

#### 3.5.2 寻路提示逻辑

- 画面横向分为 **左、中、右** 三个区域
- 检测到外卖/障碍物时，按所在区域播报方向
- 距离估算：根据检测框大小粗略判断（框越大越近）
  - 小框 → "远处"
  - 中框 → "约X米"
  - 大框 → "就在眼前"

#### 3.5.3 语音播报规则

- **优先级**：避障 > 外卖位置 > OCR结果
- **节流控制**：
  - 相同方向的避障提示，3秒内不重复
  - 外卖位置播报，2秒内不重复
  - OCR结果完整播报，不节流
- **播报语言**：中文

### 3.6 交互方式（MVP）

| 操作 | 触发方式 | 功能 |
|------|---------|------|
| 识别外卖单 | 空格键 / 说"识别" | 触发OCR识别当前画面 |
| 重复播报 | R键 / 说"重复" | 重复上一次播报内容 |
| 切换模式 | M键 | 切换检测模式（外卖/通用） |
| 退出 | Q键 / ESC | 退出程序 |

### 3.8 关键性能指标

| 指标 | 目标值 |
|------|--------|
| 检测帧率 | ≥ 15 FPS |
| 外卖检测准确率 | ≥ 70%（通用模型，后续微调提升） |

## 4. 第二阶段（移动端）规划

### 4.1 目标
将第一阶段验证通过的算法移植到Flutter移动端，产出可安装的安卓APK。

### 4.2 技术选型（初步）

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | Flutter | 跨平台，一套代码安卓/iOS |
| 物体检测 | tflite_flutter + YOLOv8n (TFLite格式) | 实时相机流检测 |
| OCR | 待定（chineseocr_lite TFLite版 / ML Kit） | 等第一阶段验证后确定 |
| 语音 | 系统TTS | 安卓自带，用户熟悉 |
| 相机 | camera 插件 | 实时帧采集 |

### 4.3 已知风险
- PaddleOCR无法直接用于Flutter/TFLite生态，移动端OCR需要换方案
- 移动端性能有限，可能需要进一步量化模型或降低分辨率
- 盲人用户的触控交互设计需要仔细打磨

---

## 5. 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| YOLO通用模型检测外卖效果差 | 高 | 中 | 收集外卖场景图片，微调模型 |
| OCR识别小票准确率低 | 高 | 中 | 尝试不同OCR模型；优化图像预处理 |
| Mac端效果好但移动端跑不动 | 中 | 中 | 模型量化、降低输入分辨率、优化推理流程 |
| 语音播报信息过载 | 中 | 高 | 做好节流和优先级控制 |
