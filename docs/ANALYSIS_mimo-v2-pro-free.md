## 项目概述

**慧眼 SmartEye** 是一款面向视障人士的 Android 端侧 AI 辅助应用，MVP 聚焦「识别外卖小票上的取餐码并语音播报」。

- **目标用户**：盲人、低视力用户
- **核心价值**：把手机摄像头对准物品，听见识别结果
- **技术栈**：Flutter + Dart + Kotlin + google_mlkit_text_recognition + 预录音频 MediaPlayer

## 可优化空间分析

### 一、代码结构优化

#### 1.1 HomeScreen 职责过重

**问题**：`lib/screens/home_screen.dart` 有 **547 行代码**，承担了多重职责：
- 相机管理与初始化
- OCR 扫描逻辑
- 手势处理（单击/双击/三击/滑动）
- 语音播报控制
- 日志管理与显示
- 历史记录播报

**建议**：
- 拆分为独立组件（CameraWidget、GestureHandler、VoiceController 等）
- 引入状态管理方案（Provider、Riverpod 或 BLoC）

#### 1.2 未使用的文件

**问题**：
- `lib/services/camera_service.dart` 文件存在但未被使用，相机逻辑直接写在 HomeScreen 中
- `lib/widgets/` 目录为空

**建议**：
- 删除未使用的 `camera_service.dart`，或将其重构为独立的相机服务
- 将可复用组件提取到 `widgets/` 目录

#### 1.3 代码重复

**问题**：
- `_isInCooldown` 逻辑在 `ocr_service.dart` 和 `home_screen.dart` 中重复实现
- 距离反馈冷却检查在 `_playScanningPrompt` 和 `_playDistanceFeedback` 中重复

**建议**：
- 统一冷却逻辑到 `OcrService` 中
- 复用 `_playDistanceFeedback` 方法，避免重复代码

### 二、性能优化

#### 2.1 OCR 扫描策略

**问题**：当前固定每 2 秒扫描一次，不够智能

**建议**：
- 动态调整扫描频率：
  - 检测到文字时加快扫描（如 1 秒）
  - 无文字时减慢扫描（如 4 秒）
- 使用图像分析判断是否需要扫描（如亮度、对比度检测）

#### 2.2 内存管理

**问题**：
- `_textRecognizer` 在 `dispose` 时关闭，但可在不需要时提前释放
- 临时文件清理逻辑可进一步优化

**建议**：
- 在 `_isAnnouncing` 为 true 时释放 `_textRecognizer`
- 使用 `try-finally` 确保临时文件一定被删除

### 三、用户体验优化

#### 3.1 手势识别

**问题**：
- 三击节奏窗口仅 600ms，对视障用户可能偏快
- 双击操作被单击吞掉（需等待 600ms 判断）

**建议**：
- 延长三击窗口至 800-1000ms
- 考虑使用「长按 + 滑动」组合替代三击
- 添加手势引导语音提示

#### 3.2 语音反馈

**问题**：
- 首次启动教程长达 8 秒，可能需要分段播放
- 缺少语音播报进度反馈
- 没有语音音量调节功能

**建议**：
- 教程分段播放，每段后有提示音
- 添加「正在识别」「识别完成」等状态语音
- 支持音量调节手势（如左右滑动）

#### 3.3 日志覆盖层

**问题**：屏幕顶部 8 行绿色日志对视障用户无意义，占用屏幕空间

**建议**：
- 正式版隐藏日志覆盖层
- 改为可开关（长按触发）
- 或仅在调试模式显示
