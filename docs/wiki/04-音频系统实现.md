# 04 - 音频系统实现详解

> 本文档详解慧眼 SmartEye 为什么放弃 flutter_tts、预录音频方案的完整实现、MethodChannel 通信机制、以及音频竞态条件的修复。这是本项目最具技术特色的模块之一。

---

## 1. 为什么放弃 flutter_tts？

### 问题发现

在 MVP 早期版本中，我们使用 `flutter_tts`（系统 TTS 引擎）进行语音播报。在大多数设备上工作正常，但在 **OPPO/ColorOS 设备**上出现严重问题：

- `flutter_tts` 调用 `await flutterTts.speak("xxx")` 没有声音
- 原生 `TextToSpeech.onInit` 回调返回 `ERROR`
- 系统设置中 TTS 引擎配置看起来正常，其他应用（如微信朗读、系统读屏）可以使用 TTS

### 根本原因

OPPO/ColorOS 等国产 ROM 对第三方应用绑定系统 TTS 引擎做了限制：
1. 系统 TTS 引擎只对系统应用和白名单应用开放完整接口
2. 第三方应用的 `TextToSpeech` 初始化静默失败（返回成功但实际不可用）
3. 这个限制没有官方文档，没有错误码，无法通过代码检测和绕过
4. 并非所有国产 ROM 都有此问题（华为、小米表现正常），但 OPPO 市场份额足够大，不能忽视

### 决策过程

我们考虑了多个替代方案：

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 修复 flutter_tts 兼容性 | 保持原有架构 | 无法绕过 ROM 限制 | ❌ 不可行 |
| 接入云端 TTS（如百度、讯飞） | 支持任意文本 | 违反隐私原则（需联网）、有延迟 | ❌ 违反端侧/隐私优先原则 |
| 引入内置 TTS 引擎（如 flutter_tts 打包模型） | 不依赖系统 | APK 体积增加 50MB+、中文支持差 | ❌ 体积不可接受 |
| **预录音频 + MediaPlayer** | 100% 兼容、无需联网、响应快、体积小 | 只能播报预录内容 | ✅ 采用 |

## 2. 音频方案概述

### 核心思路

将所有需要播报的语音拆分为**最小可拼接单元**，录制为独立音频文件。运行时根据需要拼接播放：

```
"美团外卖 65 号" → meituan.mp3 + num_6.mp3 + num_5.mp3 + hao.mp3
"井 15"          → jing.mp3 + num_1.mp3 + num_5.mp3
"没有识别到取餐码" → none.mp3（完整一句，不拆分）
```

### 音频素材清单

| 类别 | 文件 | 内容 | 大小 |
|------|------|------|------|
| 数字 | `num_0.mp3` ~ `num_9.mp3` | 零到九 | 5-8 KB/个 |
| 取餐码符号 | `jing.mp3` / `hao.mp3` | 井 / 号 | 4 KB/个 |
| 外卖平台 | `meituan.mp3` / `eleme.mp3` / `jingdong.mp3` / `taobao.mp3` / `pupu.mp3` / `yonghui.mp3` | 各平台名 | 5 KB/个 |
| 反馈音 | `di_tigao.mp3` / `di_tidi.mp3` | 距离提示"滴"声 | 1 KB/个 |
| 固定提示 | `prefix.mp3` / `none.mp3` / `help.mp3` / `scanning.mp3` 等 | 完整短语 | 5-15 KB/个 |
| 教程 | `tutorial.mp3` | 首次启动完整教程 | 30 KB |
| 历史序号 | `hist_1.mp3` ~ `hist_5.mp3` | 第1条~第5条 | 4 KB/个 |

**总体积**：所有音频素材合计约 150KB，对 APK 体积影响可忽略。

### 音频格式

- **实际格式**：MP4/AAC 容器（通过 macOS `afconvert` 生成）
- **扩展名**：`.mp3`（历史约定）
- **原因**：Android `MediaPlayer` 按文件头（`ftypmp42`）解析格式，不按扩展名判断，所以扩展名不影响播放
- **参数**：单声道、44.1kHz、AAC 编码、2.0 倍速（在原生层设置）

## 3. 音频生成方法

使用 macOS 自带工具 `say` + `afconvert` 生成，无需真人录音。

### 快速生成单条音频

```bash
# 1. 用 say 生成 AIFF（使用 Tingting 普通话女声）
say -v Tingting -o /tmp/meituan.aiff "美团外卖"

# 2. 试听
afplay /tmp/meituan.aiff

# 3. 转换为 MP4/AAC（扩展名用 .mp3）
afconvert -f mp4f -d aac /tmp/meituan.aiff assets/audio/meituan.mp3

# 4. 验证文件头（应显示 ftypmp42）
head -c 16 assets/audio/meituan.mp3 | xxd
```

### 批量生成脚本

项目提供了 `scripts/generate_audio.sh` 用于批量生成：

```bash
./scripts/generate_audio.sh "美团外卖" "饿了么" "京东外卖"
```

### 文件名规范

| 类别 | 命名规则 | 示例 |
|------|---------|------|
| 数字 | `num_<数字>` | `num_3.mp3` |
| 单字 | 拼音 | `jing.mp3`, `hao.mp3` |
| 短语 | 拼音下划线 | `faxian_waimai.mp3` |
| 平台 | 拼音/英文ID | `meituan.mp3`, `eleme.mp3` |
| 教学 | 英文描述 | `tutorial.mp3`, `help.mp3` |

### 新增语音提示的完整流程

1. 写文案（单条 ≤ 5 秒，不带语气词）
2. 用 `say` + `afconvert` 生成音频文件
3. 放入 `assets/audio/` 目录
4. 在 `pubspec.yaml` 的 `assets:` 下注册（注意：当前已配置为 `assets/audio/` 整个目录，新文件自动包含）
5. 在 `TtsService._mapTextToAssets()` 中添加映射规则
6. 添加对应的 `speakXxx()` 方法
7. 写单元测试验证路径映射
8. 构建 APK 安装到真机，听感验证语速、音量、停顿

## 4. 三层架构

音频系统从上层到下分为三层：

```
┌─────────────────────────────────────────────────┐
│   TtsService (lib/services/tts_service.dart)     │
│   职责：文本→音频片段映射、互斥锁、专用播报方法    │
├─────────────────────────────────────────────────┤
│   AudioService (lib/services/audio_service.dart) │
│   职责：MethodChannel 封装、Completer 等待回调    │
├─────────────────────────────────────────────────┤
│   MainActivity.kt (原生层)                       │
│   职责：MediaPlayer 顺序播放、AssetManager 读取  │
└─────────────────────────────────────────────────┘
```

### 4.1 TtsService 层

**核心数据结构**：`_playbackMutex`（互斥锁）

```dart
// 互斥锁防止音频重叠（CVE-STYLE-018）
Future<void>? _playbackMutex;

Future<void> speak(String text) async {
  // 1. 等待上一次播放完成（或被stop）
  if (_playbackMutex != null) {
    await stop();
  }
  
  // 2. 创建新的播放任务
  final completer = Completer<void>();
  _playbackMutex = completer.future;
  
  try {
    // 3. 文本映射为音频片段路径
    final assets = _mapTextToAssets(text);
    
    // 4. 交给 AudioService 播放
    await _audioService.playAssets(assets);
  } finally {
    completer.complete();
    _playbackMutex = null;
  }
}
```

**文本映射算法**（`_mapTextToAssets`）：

按优先级依次匹配：
1. 固定长提示（教程、无结果、帮助等）→ 单个完整音频文件
2. 距离反馈码（内部使用，非用户文本）→ beep 音
3. 取餐码（动态拼接）：
   - 提取平台关键词（美团/饿了么等）→ 对应平台音频前缀
   - 提取数字部分 → 逐位映射到 `num_X.mp3`
   - 添加后缀 "号" → `hao.mp3`
4. 兜底：无匹配则尝试按数字逐位拼接

### 4.2 AudioService 层

封装 MethodChannel 通信，处理异步等待：

```dart
MethodChannel _channel = const MethodChannel('com.smart_eye/audio');
Completer<void>? _playbackCompleter;

Future<bool> playAssets(List<String> assetPaths, {double volume = 1.0}) async {
  // 1. 确保 handler 只注册一次
  _ensureHandlerSet();
  
  // 2. 创建 Completer 等待原生回调
  _playbackCompleter = Completer<void>();
  
  // 3. 调用原生方法
  await _channel.invokeMethod('playAssets', {
    'paths': assetPaths,
    'volume': volume,
  });
  
  // 4. 等待原生 onPlaybackComplete 回调
  // 30秒安全超时，防止回调丢失
  try {
    await _playbackCompleter.future.timeout(
      const Duration(seconds: 30),
    );
    return true;
  } catch (e) {
    await stop();
    return false;
  }
}

// 原生回调处理
void _onMethodCall(MethodCall call) async {
  if (call.method == 'onPlaybackComplete') {
    _playbackCompleter?.complete();
  }
}
```

### 4.3 Kotlin 原生层

核心实现 `MainActivity.kt` 中的音频播放逻辑：

```kotlin
// 递归顺序播放音频片段
private fun playNext(paths: MutableList<String>, volume: Float) {
    if (paths.isEmpty()) {
        // 全部播完，回调 Dart
        isPlaying.set(false)
        handler.post {
            methodChannel.invokeMethod("onPlaybackComplete", null)
        }
        return
    }
    
    val path = paths.removeAt(0)
    try {
        // ⚠️ 关键陷阱：AssetManager 路径必须加 flutter_assets/ 前缀！
        val afd = assets.openFd("flutter_assets/$path")
        
        currentPlayer = MediaPlayer().apply {
            setDataSource(afd.fileDescriptor, afd.startOffset, afd.length)
            afd.close()
            
            // 标记为无障碍语音，系统会调整音频焦点
            setAudioAttributes(
                AudioAttributes.Builder()
                    .setUsage(AudioAttributes.USAGE_ASSISTANCE_ACCESSIBILITY)
                    .setContentType(AudioAttributes.CONTENT_TYPE_SPEECH)
                    .build()
            )
            
            setVolume(volume, volume)
            
            // 2.0 倍速播放
            playbackParams = PlaybackParams().apply { speed = 2.0f }
            
            setOnCompletionListener {
                release()
                currentPlayer = null
                playNext(paths, volume)  // 递归播放下一个
            }
            
            setOnErrorListener { _, what, extra ->
                release()
                currentPlayer = null
                playNext(paths, volume)  // 单片段失败不中断，继续下一个
                true
            }
            
            prepare()
            start()
        }
    } catch (e: Exception) {
        // 单片段打开失败，继续下一个
        playNext(paths, volume)
    }
}
```

## 5. 关键陷阱与解决方案

### 陷阱 1：flutter_assets 路径前缀

**问题**：Dart 中使用 `assets/audio/num_1.mp3` 作为 asset key，但打包到 APK 后，实际路径是：

```
apk/assets/flutter_assets/assets/audio/num_1.mp3
```

Android `AssetManager.openFd()` 需要从 APK 根目录开始的路径，所以必须加 `flutter_assets/` 前缀：

```kotlin
// ❌ 错误：找不到文件，抛 FileNotFoundException
assets.openFd("assets/audio/num_1.mp3")

// ✅ 正确
assets.openFd("flutter_assets/assets/audio/num_1.mp3")
```

**教训**：Flutter 的 asset key 和 Android AssetManager 的路径不是一回事，中间多了一层 `flutter_assets/`。

### 陷阱 2：音频重叠（竞态条件）

**问题**：早期版本中，`speak()` 是 fire-and-forget 模式，连续调用两次 `speak()` 会导致两个 MediaPlayer 同时播放，声音重叠嘈杂。

**错误修复尝试**：
```dart
// ❌ 错误：用 delay 等待播放完成
await _ttsService.speak("第一句");
await Future.delayed(const Duration(seconds: 2));  // 猜测播放时长
await _ttsService.speak("第二句");
```

为什么错误？音频时长不确定（数字位数不同、平台名长度不同），2秒可能不够也可能太长。

**正确修复（CVE-STYLE-018）**：
1. Dart 层使用互斥锁 `_playbackMutex`
2. 每次 `speak()` 前先 `await stop()`
3. 原生层 `stopPlayback()` 真正 stop + release 掉 `currentPlayer`，不仅是设标志位
4. 使用 `onCompletion` 回调精确等待播放完成，不做时间估算
5. 原生层用 `AtomicBoolean isPlaying` + `@Volatile currentPlayer` 保证多线程可见性

### 陷阱 3：扩展名与实际格式不匹配

**问题**：文件扩展名是 `.mp3`，但实际是 MP4/AAC 格式。

**为什么这样做**：
- 早期用 `afconvert` 生成时误用了 `.mp3` 扩展名
- Android MediaPlayer 按文件头解析，不看扩展名，所以能正常播放
- 如果改成 `.m4a` 或真 MP3，需要修改 TtsService 中所有路径映射，风险大
- 保持现状即可，不影响功能

**注意**：如果将来替换为真人录音，建议统一为 `.m4a` 或 `.mp3` 真实格式，并批量更新路径。

### 陷阱 4：音频焦点

Android 系统有音频焦点机制，当其他应用播放声音（如导航、来电）时，应该暂停或降低音量。

当前实现：设置 `USAGE_ASSISTANCE_ACCESSIBILITY`，系统会将其视为无障碍语音，优先级较高，不会被普通媒体声音打断。这对于视障应用是合理的——语音播报是用户获取信息的唯一途径，不应被轻易打断。

### 陷阱 5：MediaPlayer 资源释放

每个音频片段使用独立的 `MediaPlayer` 实例，在 `onCompletionListener` 中必须调用 `release()` 释放资源，否则会导致内存泄漏和后续播放失败。

```kotlin
setOnCompletionListener { mp ->
    mp.release()  // 必须释放！
    currentPlayer = null
    playNext(paths, volume)
}
```

## 6. 扩展指南

### 如何新增一个固定语音提示

1. 生成音频文件，放入 `assets/audio/`
2. 在 `TtsService` 中添加方法：
```dart
Future<void> speakLowBattery() async {
  await speak('low_battery');  // 会被 _mapTextToAssets 映射
}
```
3. 在 `_mapTextToAssets()` 中添加匹配规则：
```dart
if (trimmed.contains('电量低')) {
  return ['assets/audio/low_battery.mp3'];
}
```
4. 在需要播报的地方调用 `await _ttsService.speakLowBattery()`
5. 写单元测试

### 如何支持动态数字朗读

数字朗读已经实现，利用 `num_0.mp3` ~ `num_9.mp3` 逐位拼接：

```dart
// "15" → num_1.mp3 + num_5.mp3
final digits = code.split('');
return digits.map((d) => 'assets/audio/num_$d.mp3').toList();
```

如果需要支持更多动态内容（如时间、百分比），需要录制更多基础单元（十、百、点、百分之等）。

### 如何切换到真人录音

1. 录制 WAV 格式音频（44.1kHz，单声道）
2. 转换格式：`afconvert -f mp4f -d aac input.wav assets/audio/xxx.mp3`
3. 替换同名文件即可，代码不需要修改
4. 建议统一录制所有数字、平台名、提示语，保持音色一致

### 如何重新引入 TTS 作为备选

如果将来需要支持任意文本朗读（如读取完整小票内容），可以采用"预录音频为主 + TTS为辅"的混合方案：

1. 尝试初始化系统 TTS
2. 如果 TTS 可用（非OPPO等受限设备），固定提示用预录音频（音色统一），动态长文本用 TTS
3. 如果 TTS 不可用，降级到纯预录音频方案
4. 保持 TtsService 的 API 不变，调用方无需修改
