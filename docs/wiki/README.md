## 核心设计原则

> 这四个原则是项目所有决策的基石，任何新功能都必须符合。

1. **无障碍优先（Accessibility First）**
   - 所有交互必须有语音反馈
   - **NSF 红线（No Silent Failure）**：任何异常路径绝不允许静默失败，必须语音告知用户发生了什么、该怎么办
   - 禁止复杂手势（双指缩放、旋转等）
   - 禁止仅依赖视觉的反馈（颜色变化、Toast、Snackbar 无语音）

2. **隐私优先（Privacy First）**
   - 纯端侧 OCR（google_mlkit_text_recognition），禁止任何云端 API
   - 不上传任何图片、帧数据、设备信息、位置信息
   - 历史记录本地加密存储

3. **端侧优先（On-device First）**
   - 所有 AI 推理在本地完成，无网络依赖也能正常工作
   - 网络仅用于应用内更新下载，且有白名单限制

4. **可靠优先（Reliability First）**
   - 语音播报是用户唯一的信息来源，必须 100% 可靠
   - 相机初始化失败、权限被拒、OCR 出错等全部要有兜底语音
