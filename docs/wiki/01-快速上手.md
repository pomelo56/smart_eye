# 01 - 快速上手指南

> 本文档帮助新开发者在 30 分钟内完成环境搭建、首次构建、并安装到真机运行。

---

## 1. 环境要求

| 工具 | 最低版本 | 说明 |
|------|---------|------|
| Flutter | 3.x | 建议使用 stable 渠道 |
| Dart | 3.x | 随 Flutter 安装 |
| Android SDK | API 21+ (Android 5.0) | targetSdkVersion 34 |
| JDK | 17 | Android Gradle Plugin 8.x 需要 |
| 操作系统 | macOS / Linux / Windows | 主要在 macOS 上开发测试 |
| 测试设备 | Android 真机 arm64-v8a | 2019 年后的 Android 设备均支持 |

> **注意**：项目仅构建 `arm64-v8a` ABI，不支持 32 位设备和 x86 模拟器。

## 2. 获取代码

```bash
# Gitee（国内推荐）
git clone https://gitee.com/free-style_2_0/smart_eye.git
cd smart_eye

# GitHub（海外）
git clone https://github.com/pomelo56/smart_eye.git
cd smart_eye
```

配置双远程仓库（推荐）：

```bash
# 已有 origin（Gitee），添加 github
git remote add github https://github.com/pomelo56/smart_eye.git

# 验证
git remote -v
# origin -> gitee
# github -> github
```

## 3. 安装依赖

```bash
flutter pub get
```

如果遇到 Flutter SDK 缓存权限问题（macOS 常见）：

```bash
sudo chown -R $(whoami) <你的 Flutter 安装目录>/bin/cache
flutter pub get
```

## 4. 首次构建

### Debug 构建（开发用）

```bash
flutter build apk --debug
# 输出：build/app/outputs/flutter-apk/app-debug.apk (~89 MB)
```

### Release 构建（发布用）

```bash
flutter build apk --release
# 输出：build/app/outputs/flutter-apk/app-release.apk (~31 MB)
```

> Release APK 使用项目自带的 keystore 自动签名，无需额外配置。

---

## 5. 安装到真机

### 方式一：USB 连接

```bash
# 检查设备
adb devices

# 安装
adb install -r build/app/outputs/flutter-apk/app-release.apk
```

### 方式二：无线 ADB 调试（推荐，无需数据线）

详细步骤参见 [`无线调试安装APK.md`](../无线调试安装APK.md)，简要流程：

1. 手机开启「开发者选项」→「无线调试」
2. 手机和电脑在同一 WiFi 网络
3. 手机点击「使用配对码配对设备」，显示 IP:端口 和配对码
4. 电脑执行：
   ```bash
   adb pair <手机IP>:<配对端口>
   # 输入配对码

   adb connect <手机IP>:<连接端口>
   adb devices  # 确认设备已连接
   ```
5. 安装：
   ```bash
   adb -s <设备IP:端口> install -r build/app/outputs/flutter-apk/app-release.apk
   ```

### 方式三：开发时直接运行

```bash
flutter run
# 自动安装 debug 版本并启动，支持 Hot Reload
```

## 6. 验证安装成功

安装后打开 App，应该能听到：
1. **首次启动**：「欢迎使用慧眼 SmartEye...」完整教程语音
2. **非首次启动**：相机预览画面（对盲人用户只有语音反馈）

基本功能验证：
- 将手机对准带有 `#15` 格式的取餐码小票
- 听到「取餐码是 井 1 5」的播报
- 单击屏幕：重听上一次结果
- 三击屏幕：重新识别
- 上滑：播报历史记录
- 下滑：播报操作帮助

## 7. 开发工作流

### 使用 Git Worktree（推荐，避免污染 main 分支）

```bash
# 从 main 创建 feature 分支和独立工作目录
git worktree add ../smart_eye_feature_xxx -b feat/xxx main

# 在新目录中开发
cd ../smart_eye_feature_xxx
flutter pub get
# ... 写代码、写测试 ...

# 开发完成后合并回 main
cd ../smart_eye
git merge feat/xxx --ff-only
git worktree remove ../smart_eye_feature_xxx
```

### TDD 开发流程

项目强制 TDD（测试驱动开发）：

```bash
# 1. 先写测试（RED）
# 在 test/unit/ 下编写测试用例

# 2. 运行测试，确认失败
flutter test test/unit/xxx_test.dart

# 3. 写实现代码（GREEN）
# 在 lib/ 下编写实现

# 4. 运行测试，确认通过
flutter test

# 5. 重构（REFACTOR）
dart format lib/ test/
flutter analyze  # 必须零警告
```

### 常用命令速查

```bash
flutter pub get              # 安装依赖
dart format lib/ test/       # 格式化代码
flutter analyze              # 静态分析（零警告）
flutter test                 # 运行所有单元测试
flutter test test/unit/      # 只跑单元测试
flutter build apk --release  # 构建 Release APK
./scripts/test.sh            # 一键测试（format + analyze + test）
./scripts/release.sh vX.Y.Z  # 一键发布（含双仓库推送）
```

## 9. 第一次开发要做什么？

1. 阅读 [02-架构设计详解](./02-架构设计.md) 理解整体架构
2. 阅读 [03-无障碍开发指南](./03-无障碍开发指南.md) 理解最核心的 NSF 红线
3. 运行 `flutter test` 确认所有测试通过
4. 构建 Release APK 并安装到真机，体验完整功能
5. 阅读 [05-Flutter-Android踩坑手册](./05-Flutter-Android踩坑手册.md) 避免重复踩坑
6. 从一个小功能入手（比如新增一条语音提示），熟悉代码结构

## 10. 常见问题

### Q: 构建时提示 Kotlin/Gradle 版本不兼容？
A: 这是环境警告，不影响 APK 产出。如需消除，可升级 Android Studio 或调整 Gradle 版本。

### Q: 安装时提示「应用未安装」？
A: v0.9.0 起包名从 `com.example.smart_eye` 改为 `com.smart_eye`，需要先卸载旧版本再安装。

### Q: 相机打开是黑屏？
A: 检查是否授予了摄像头权限。首次启动应语音提示权限请求。

### Q: 没有语音播报？
A: 检查媒体音量是否打开。语音走的是媒体音量通道，不是通话音量。

### Q: flutter analyze 有警告怎么办？
A: 项目要求零警告，必须修复所有警告才能提交。
