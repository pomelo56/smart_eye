## 1. 项目概述

### 1.1 项目名称
**慧眼 SmartEye** — AI 视障寻物助手

### 1.2 背景
视障人士在日常生活中面临大量「看不见、找不到」的困难：
- 在集体取餐点找不到自己的外卖
- 无法读取外卖盒上的取餐码
- 在室内找不到日常物品

### 1.3 产品定位
面向视障人群的**端侧 AI 寻物辅助工具**。用户打开摄像头对准物品，AI 实时识别并通过语音播报结果，帮助视障用户**「听见物品、找到东西」**。

### 1.4 核心原则
- **无障碍优先**：所有功能必须通过语音反馈，不依赖视觉
- **隐私优先**：所有图像处理本地完成，不上传云端
- **可靠优先**：核心逻辑必须 TDD 覆盖，确保识别准确率
- **端侧优先**：全部 AI 推理在设备本地运行，不依赖网络

## 2. 用户故事

### US-01：识别取餐码（MVP 核心场景）
> 作为视障用户，我将手机摄像头对准外卖袋上的打印小票，应用自动识别取餐码并用语音播报，以便我知道取餐码是多少。

**验收标准：**
- 美团外卖小票上的取餐码（格式 `#` + 1-3 位数字，如 `#15`）识别率 >= 95%
- 多帧验证：连续 2 次扫描结果一致才播报（降低误识别）
- 扫描间隔 2 秒，播报后 5 秒内相同结果去重
- 播报内容："取餐码是 井 15"
- 距离反馈音：检测到文字区域时播放缓慢提示音，可识别时播放快速提示音（类似倒车雷达）

### US-02：手势操作与重新识别
> 作为视障用户，我听到识别结果后可以通过简单手势重听或重新识别，以便纠正可能的误识别。

**验收标准：**
- 单击屏幕：**重听**上一次播报的取餐码
- 三击屏幕：**重新识别**（清除当前结果，重新开始扫描）
- 识别结果**自动保存**到历史记录，无需手动确认
- 每次操作都有语音反馈

### US-03：语音播报历史与帮助
> 作为视障用户，我可以听到最近识别过的记录，以及在需要时获取操作帮助。

**验收标准：**
- 向上滑动：播报最近识别记录（最多 5 条）
- 向下滑动：播报操作帮助（"单击重听，三击重新识别，上滑历史"）
- 每条记录包含：识别内容 + 识别时间
- 历史记录保留 24 小时，超过自动清除

## 3. 功能范围

### 3.1 MVP 功能（v0.1.0）
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 相机实时预览 | P0 | 打开应用即启动相机，锁定竖屏 |
| OCR 持续扫描 | P0 | 每 2 秒扫描一次，多帧验证（连续 2 次一致才播报） |
| TTS 语音播报 | P0 | 正常语速，扬声器最大音量 |
| 距离反馈音 | P0 | 检测到文字时缓慢提示音，可识别时快速提示音 |
| 手势操作 | P0 | 单击重听、三击重识、上滑历史、下滑帮助 |
| 历史记录 | P1 | 保留 24 小时，超过自动清除 |
| 首次启动教程 | P1 | 首次打开自动播报完整操作指南 |
| 运行日志 | P1 | 本地运行日志，用于排查问题 |

### 3.2 后续迭代方向
| 功能 | 说明 | 阶段 |
|------|------|------|

### 4.3 数据流

```
用户打开应用 → 相机自动启动 → 首次启动播报操作教程
                    ↓
            持续扫描（每 2 秒一次）
                    ↓
            检测到文字区域 → 播放距离反馈提示音
                    ↓
            OCR 识别文本 → 正则提取 # + 数字
                    ↓
            多帧验证（连续 2 次一致）
                    ↓
            TTS 播报识别结果（5 秒冷却去重）
                    ↓
            自动保存到历史记录
                    ↓
            用户单击重听 / 三击重新识别
```

## 5. 非功能需求

### 5.1 性能
- 相机启动时间 <= 2 秒
- OCR 识别延迟 <= 500ms（在主流 Android 设备上）
- TTS 播报延迟 <= 200ms

### 5.2 兼容性
- **最低 Android 版本**：Android 7.0（API 24）
- **目标设备**：中端及以上 Android 手机
- **屏幕方向**：锁定竖屏
- **应用名称**：慧眼

### 5.3 无障碍
- 所有交互必须通过语音反馈
- 支持 Android TalkBack
- 所有 Widget 必须有 Semantics label
- 手势操作不依赖视觉定位

### 5.4 隐私
- 图像数据不上传
- AI 推理全部本地运行
- 不收集用户个人信息
- 不请求非必要权限

## 7. 风险与待确认事项

| 风险/待确认 | 影响 | 应对方案 |
|------------|------|---------|
| 取餐码格式（已确认美团为 `#` + 数字） | 中 | MVP 聚焦美团，其他平台后续扩展 |
| 低端机 OCR 性能差 | 中 | 最低 Android 7.0，识别失败时语音提示 |
| 光线条件影响识别 | 中 | 画面过暗时语音提示「光线不足，请打开手机手电筒」 |
| 多目标场景 | 中 | 检测到多个取餐码时，从左到右依次播报（最多 5 个） |
| TTS 中文引擎缺失 | 中 | 启动时检测，缺失则引导用户安装中文语音包 |
| 手势误触发 | 低 | 自然手抖不太可能连续三击，误触率可控 |
