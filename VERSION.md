# 版本管理规范

> 慧眼 SmartEye 采用语义化版本（Semantic Versioning）+ git tag 管理版本。

## 版本号格式

`MAJOR.MINOR.PATCH+BUILD`

| 字段 | 含义 | 何时递增 |
|------|------|---------|
| MAJOR | 不兼容的架构变更 | 大重构、技术栈更换 |
| MINOR | 新功能，向下兼容 | 新增平台、新交互、新语音 |
| PATCH | Bug 修复，向下兼容 | 修 OCR 误识别、修音频时序 |
| BUILD | 构建号（整数） | 每次发布递增 |

## 发布流程

每次迭代完成后：

1. 更新 `pubspec.yaml` 的 `version` 字段
2. 更新 `CHANGELOG.md`，将 `[Unreleased]` 归档为版本号
3. 更新本文件的「当前版本」和「版本历史」
4. 提交：`git commit -m "release: vX.Y.Z"`
5. 打标签：`git tag vX.Y.Z`
6. 推送：`git push && git push --tags`

## 回滚方法

```bash
# 查看所有版本
git tag -l

# 回滚到某个版本（保留工作区）
git checkout vX.Y.Z -- .

# 回滚到某个版本（完整重置）
git reset --hard vX.Y.Z
```
